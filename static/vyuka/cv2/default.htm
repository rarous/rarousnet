Cvičení 2

Dnes jsme si ukázali jak vypadají soubory XML (eXtensible Markup Language). Řekli jsme si, že se skládají z elementů, které jsou tvořeny tagy, atributy a obsahem, jehož součástí mohou být další elementy, text a entity, nebo může být element prázdný a pak se uzavírá samouzavírací závorkou. Každý element musí mít pouze jeden tag, ale může mít více atributů [0 - n].

<tag atribut="obsah_atributu">obsah elementu</tag>
<tag atribut="obsah_atributu" />  <!-- Samozavírací element -->
&entita;

Řekli jsme si, že všechny tagy, atributy a entity jsou předepsány pro daný dokument pomocí DTD. DTD je uvedeno na začátku dokumentu pomocí příkazu <!DOCTYPE ... Většinu používaných dokumentů na webu standardizovalo sdružení W3C, které se stará o vývoj webu. Více informací naleznete na http://w3.org.

<?xml version="1.0" ?>
<adresa>
    <jmeno typ="křestní">Karel</jmeno>
    <jmeno typ="příjmení">Novák</jmeno>
    <ulice cp="110">Liberecká</ulice>
    <obec psc="46352">Osečná</obec>
</adresa>

Dále jsme si vytvořili jednoduchý ukázkový XML soubor, ve kterém jsou různé elementy s atributy a textem. Ukázku jsme si uložili jako soubor adresa.xml a prohlédli v prohlížeči, kde bylo vidět, že jsou jednotlivé elementy v dokumenty uspořádány do stromu. Potom jsme si prohlíželi strukturu složitějšího XML souboru na mých stránkách, šlo o vizitku ve formátu RDF.

HTML

Do HTML jsme se dnes naučili vkládat obrázky a odkazy na jiné dokumenty. Obrázek se vkládá pomocí tagu img a atributů src a alt. Atribut src určuje adresu obrázku na internetu a atribut alt obsahuje text, který se zobrazí místo obrázku pokud není dostupný. Odkaz se vytváří pomocí tagu a a atributu href, který určuje cílovou adresu odkazu. V elementu odkazu může být jako obsah vložen obrázek nebo text.

<img src="http://osecna.aspweb.cz/galerie/okoli/img00018.jpg" alt="foto osečné" />
<a href="http://seznam.cz">Seznam.cz</a>
<a href="http://osecna.obec.cz">
    <img src="http://osecna.aspweb.cz/galerie/okoli/img00018.jpg" alt="foto osečné" />
</a>
